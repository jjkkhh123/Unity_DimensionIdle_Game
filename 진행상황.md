## 🎮 현재 게임 시스템 상세

### 차원 시스템
- **1~8차원 존재**
- 각 차원은 하위 차원을 생산
- 1차원은 반물질 생산
- **초기 해금**: 1~2차원
- **해금 조건**: **Dimension Enhance(DimBoost)를 통해서만 해금** ⭐ 변경됨

### 가격 시스템 (2025-12-17 업데이트)

**초기 가격**:
- 1차원: 10
- 2차원: 1,000
- 3차원: 1e10
- 4차원: **1e40** ⭐ (이전: 1e30)
- 5차원: **1e80** ⭐ (이전: 1e100)
- 6차원: **1e130** ⭐ (이전: 1e200)
- 7차원: **1e200** ⭐ (이전: 1e300)
- 8차원: **1e260** ⭐ (이전: 1e400)

**가격 증가**:
- 1개 구매마다: **20% 증가 (1.20배)** ⭐ (이전: 15%)
- 10개 구매마다: 5배 추가 증가
- 10개 구매 시 가격 배수가 15%씩 증가 (5배 → 5.75배 → 6.6125배...)

### Dimension Enhance (DimBoost) 시스템 ⭐ NEW

**조건 (8차원 해금 전)**:
- 현재 최고 해금 차원을 **20개 구매**
- 3차원 이상 해금되어 있어야 함

**조건 (8차원 해금 후)**: ⭐ 최신 업데이트
- **8차원 20개** → DimBoost 가능
- **8차원 40개** → DimBoost 가능
- **8차원 60개** → DimBoost 가능
- **무제한 반복 가능!** (20개씩 증가)

**효과**:
1. **안티매터 10으로 리셋**
2. 모든 차원 리셋 (수량, 구매 횟수)
3. Tickspeed 리셋
4. **다음 차원 해금** (8차원 전까지만)
5. **차원별 누적 배수 적용**:
   - 1회: 1차원만 ×2
   - 2회: 1~2차원 각각 ×2 (1차원 총 ×4, 2차원 총 ×2)
   - 3회: 1~3차원 각각 ×2 (1차원 총 ×8, 2차원 총 ×4, 3차원 총 ×2)
   - 8회 이상: **모든 차원 ×2** (무제한 성장!)

**예시**:
```
3차원 20개 구매 → DimBoost 1회 → 4차원 해금 + 1차원 ×2
4차원 20개 구매 → DimBoost 2회 → 5차원 해금 + 1~2차원 ×2
5차원 20개 구매 → DimBoost 3회 → 6차원 해금 + 1~3차원 ×2
6차원 20개 구매 → DimBoost 4회 → 7차원 해금 + 1~4차원 ×2
7차원 20개 구매 → DimBoost 5회 → 8차원 해금 + 1~5차원 ×2

8차원 해금 후:
8차원 20개 구매 → DimBoost 6회 → 모든 차원 ×2
8차원 40개 구매 → DimBoost 7회 → 모든 차원 ×2
8차원 60개 구매 → DimBoost 8회 → 모든 차원 ×2
... 무한 반복!
```

**UI 표시**:
- 8차원 해금 전: "Unlock Xth Dimension"
- 8차원 해금 후: "Boost All Dimensions"
- 조건 표시: "Requires: X 8th Dimensions / Current: Y"

## 🐛 수정된 버그 기록

### DimBoost 배수 누적 버그 (2025-12-18 수정) ⭐ 중요
- **증상**: 프레스티지 후에도 DimBoost 배수가 초기화되지 않고 계속 누적됨
  - 예: DimBoost 1회(×2) → 프레스티지 → DimBoost 1회 → 잘못된 ×4 (올바른 값: ×2)
- **원인**: `Dimension.Reset()` 함수에서 `multiplier` 필드를 초기화하지 않음
- **해결**: `Dimension.cs:121`에 `multiplier = BigDouble.One;` 추가
- **결과**: 프레스티지 후 DimBoost 배수가 올바르게 초기화됨 ✅

### 프레스티지 상점 가격 표시 버그 (2025-12-17 수정)
- **증상**: UI에 표시된 가격보다 많은 포인트가 차감됨
- **원인**: UI는 `upgrade.cost` (초기 가격), 실제는 `GetNextCost()` (레벨별 증가 가격)
- **해결**:
  - `PrestigeShopUI.cs:175` - 버튼 생성 시 `GetNextCost()` 사용
  - `PrestigeShopUI.cs:231` - UI 업데이트 시 실시간 가격 반영
- **결과**: UI 표시 가격 = 실제 차감 가격 ✅
